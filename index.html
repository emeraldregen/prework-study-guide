HTML

  • Usually composed of an opening tag and closing tag, angle brackets (< >).
  • HyperText Markup Language-most basic building block. Handles the content, structure, and meaning of each element.
  • The head element contains information/metadata about the webpage.
  • Title-required for Search Engine Optimization,
  • Style-defines css
  • Base-defines url
  • Link-connects external resource
  • Meta-character set,description, keywords, author, viewport
  • Scripts-embeds client-side scripts.
  • The body element represents the visible content shown to the user.
  • ul, ol, and li elements represent unordered lists, ordered lists, and list items.
  • br element creates a line of empty space, or a line break
  • a element (which stands for anchor) creates links to the same webpage or other webpages. The href attribute points to the URL for the link,
  • src attribute defines the location, or source, of the image file.
  • alt attribute contains a text string that describes the appearance and functionality of the image.

CSS

  • Cascading Style Sheets-describe presentation of a website and how each element is rendered/styled.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.

Git

  • free, open source version control system that allows web developers to collaborate on projects without overwriting each other's work.
  • Allows developers to revert to previous versions of the code if a problem is discovered.
  • git status: checks what branch we are currently on
  • git checkout name: changes the branch you're on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -A: commit all files in the working branch
  • git commit -m "message": to associate a message with our commit. Commit messages should be short descriptive messages of the changes implemented.
  • git pull origin main: makes sure current branch is up to date with base branch before pushing up any code!
  • git push origin feature/name: Push changes, then able to merge feature with base on github

JavaScript

  • Provides logic and user interactivity.
  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
